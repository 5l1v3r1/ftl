Detailed Description

template<typename T, typename A>
struct ftl::monad< std::list< T, A > >

Monad implementation for std::list.

Behaves like most collections; which is to say, the monad can be said to model nondeterministic computations.

Applies f to each element.

The result of each call to f is collected and returned as an element in the list returned by the call to map.

Example:

// l == list<int>{2,3,4}

static

Monad bind operation.

Can be viewed as a non-deterministic computation: l is a list of possible values, each of which we apply f to. As f itself is also non-deterministic, it may return several possible answers for each element in l. Finally, all of the results are collected in a flat list.

Note
f is allowed to return any ForwardIterable, not only lists. The final result, however, is always a list.

Example:

auto l =
list<int>{1, 2, 3}
>>= [](int x){ return list<int>{x-1, x, x+1}; };
// Note how each group of three corresponds to one input element
// l == list<int>{0,1,2, 1,2,3, 2,3,4}
