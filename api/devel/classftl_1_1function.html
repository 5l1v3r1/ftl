Detailed Description

template<typename, typename...>
class ftl::function< typename, >

Function object encapsulator.

Template Parameters
RReturn value of the wrapped function or function object.
PsParameter pack of the wrapped function's operator().
Concepts
  • FullyConstructible
  • Assignable
  • Function<R(Ps...)>
  • Function<Function<R(P2, ..., PN)>(P1)>, assuming sizeof...(Ps) > 2 and P1, P2, ..., PN are the elements of Ps.
  • Applicative Functor
  • Functor
  • Monoid, if and only if R is a Monoid.

If the second Function instance above is not completely obvious in what it means, it signifies that ftl::function supports the notion of curried calling. For example:

ftl::function<int,int,int> f = ... // Binary function of integers
f(1,2); // Call directly
auto g = f(1); // Curried call, providing one parameter only
g(2); // Results in the same as f(1,2);
Note
While often convenient, it may give rise to ambiguous calls to operator() if the function object encapsulated by the ftl::function has several overloaded operator() that differ only in the number of parameters they take.
Warning
Curried calling will result in copies of the parameter being made. Every time you invoke operator() without filling the complete parameter list, you are creating copies.
