Detailed Description

template<template< typename...> class F>
interface ftl::functor< F >

Struct that must be specialised to implement the functor concept.

When specialising, it is important to remember that the described interface here is minimal, and an instance is encouraged to include additional overloads of the functions for optimization reasons and similar. For types that cannot be copied, for instance, it makes sense that all of the functor interface functions accept an rvalue reference.

Note
There exists a second functor interface struct, declared as
template<template<typename> class F>
struct functor<F> { ... };
The reason for this is that it is presently impossible to unify the two signatures, and FTL must allow functors to be parameterised on more than just the type they're a functor in. Each member of the interface will further document how the second version looks.

Maps a function to the contained value(s).

For instances parameterised on only A, map's signature is

static F<B> map(Fn&& fn, const F<A>& f);

where the template parameters A, B, and Fn are declared equivalently to this version.
