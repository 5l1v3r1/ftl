Detailed Description

template<typename N>
struct ftl::prod_monoid< N >

Implementation of monoid for numbers, when interpreted as products.

The reason behind this struct is exactly the same as with ftl::sum_monoid.

The behaviour of the product monoid is simple:

id() => N(1)
operator^ => N::operator*
Template Parameters
NAny integer or floating point primitive type, or a type that implements operator* (in a way that does not violate the monoid laws) and can be constructed from the literal 1.
Concepts
  • FullyConstructible
  • Eq, if N is
  • Orderable, if N is
Examples

Trivial usage:

float foo() {
using ftl::operator^;
ftl::prod_monoid<float> x = 3.f, y = 2.5f;
return x ^ y ^ ftl::monoid<ftl::prod_monoid<float>>::id();
}

foo() will always return the result of 3.f*2.5f*1.f.

Use the fact that "monoidness" is transitive in the case of ftl::maybe and its contained type:

ftl::maybe<ftl::prod_monoid<int>> m1{2};
ftl::maybe<ftl::prod_monoid<int>> m2{3};
ftl::maybe<ftl::prod_monoid<int>> m3 = ftl::mappend(m1, m2);
if(m3) std::cout << *m3;

Outputs "6"
