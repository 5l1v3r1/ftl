Detailed Description

template<typename M_>
struct ftl::deriving_bind< back_insertable_container< M_ > >

Inheritable bind implementation for containers supporting insert.

The following requirements must be met by M_:

  • There must exist an M::insert(iterator pos, const T& value), returning an iterator to the inserted element. An R-value version is also beneficial in some cases.
  • There must be an implementation of monad::map available for M_ that is not derived in terms of bind.

Also note that types using this construct to generate a bind implementation will be capable of binding with any function returning a ForwardIterable, regardless of which ForwardIterable that is. I.e., in the example below, you could bind MyContainer with a function returning e.g. maybe<T>, std::vector<T>, etc.

Example:

namespace ftl {
template<typename T>
struct monad<MyContainer<T>>
: deriving_bind<back_insertable_container<MyContainer<T>> {
// Implementations of pure, map, join, apply
};
}

This bind version follows the standard container theme of modelling non-deterministic computations.

In other words, m is viewed as a collection of possible values, each of which `f' will generate a new set of possible answers for.

Examples
SomeCollection<int> c{2,4,5};
auto c2 = c >>= [](int x){ return SomeCollection<int>{x/2, 2*x}; };
// Note how each group of two results correspond to one input value
// c2 == SomeCollection<int>{1,4, 2,8, 2,10}
