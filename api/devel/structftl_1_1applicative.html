Detailed Description

template<typename F>
interface ftl::applicative< F >

Struct that must be specialised to implement the applicative concept.

Member Typedef Documentation

template<typename F >
using ftl::applicative< F >::T = concept_parameter<F>

The type F is an applicative functor on.

For example, in the case of maybe<int>, T = int.

static F ftl::applicative< F >::pure ( T &&  x)
static

Encapsulate a pure value in the applicative functor.

Defaults to monad<F>::pure, because any monad is also an applicative functor.

Note
Default implementation only works if F has a monad instance defined.
If implementing applicative directly (instead of implicitly by way of monad), then pure may be overloaded or replaced with a const reference version, or even a pass by value version.

Contextualised function application.

Applies the wrapped/contextualised function fn to the similarly wrapped value f.

Default implementation is to use monad's ap().
