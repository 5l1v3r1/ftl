template<typename T>
struct ftl::parametric_type_traits< T >

Traits for various parametric types.

In most cases, the default should suffice, but it is possible to specialise if required.

template<typename T >
using ftl::parametric_type_traits< T >::value_type = typename inner_type<T>::type

Gets the type parameter of T on which concepts apply.

Many of the concepts in FTL are parametric in one type, e.g. ftl::functor<ftl::maybe<U>> is parametrised by U. This type level function should return that type.

The only time this trait needs to be specialised is when the type parameter T uses in concepts is not the left-most in T's parameter list.

Example demonstrating semantics:

// x is of type int
typename parametric_type_traits<either<string,int>::value_type x;
template<typename T >
template<typename U >
using ftl::parametric_type_traits< T >::rebind = typename _dtl::default_rebind<T,U>::type

Specialisable type function to rebind a parametric type.

The purpose of this type function is to swap the value_type of a parametric type. This is useful when writing templated functions that deal with genericised parameters, such as a function from a functor<T> to functor<U> (i.e., the concrete types could be maybe<int> and maybe<float> for some instantiation).

Example:

// x could be e.g. list<int>, if M is a list of some type T
parametric_type_traits<M>::template rebind<int> x;
