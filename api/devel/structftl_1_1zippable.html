Detailed Description

template<typename Z_>
interface ftl::zippable< Z_ >

Concrete definition of the zippable concept.

The most typical types that are instances of zippable are containers of different sorts, in which case the zip operation simply iterates one element at a time from each container and "zips" them together, commonly with the tuple constructor.

Note that any applicablepg can be made zippable with the following zipWith implementation:

auto zipWith(F f, const A& a, const A& b) {
// aapply(fmap(f, a), b)
return f % a * b;
}
