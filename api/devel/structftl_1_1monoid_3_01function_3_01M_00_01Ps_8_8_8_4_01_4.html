Detailed Description

template<typename M, typename... Ps>
struct ftl::monoid< function< M, Ps...> >

Monoid instance for ftl::functions returning monoids.

The reason this works might not be immediately obvious, but basically, any function (regardless of arity) that returns a value that is an instance of monoid, is in fact also a monoid. It works as follows:

id() <=> A function, returning monoid<result_type>::id(),
regardless of parameters.
append(a,b) <=> A function that forwards its arguments to both a and b,
and then calls monoid<result_type>::append on the two
results.

Compute f1 and f2 and combine the results with monoid operation.

Note
The parameter pack Ps cannot be perfectly forwarded, because both f1 and f2 must be called with the same parameters. It is therefore recommended you use this monoid operation only with functions working on values or const references.
Warning
Should e.g. f1 mutate one of the parameters, it will affect the call to f2. Be very careful if either f1 or f2 has side effects.
