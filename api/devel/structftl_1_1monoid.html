Detailed Description

template<typename M>
interface ftl::monoid< M >

Concrete monoid interface.

For any type to be an instance of the monoid concept, it must specialise this interface.

Member Function Documentation

template<typename M >
static M ftl::monoid< M >::id ( )
static

Get the identity element for a given monoid.

Which element to be used depends on what operation the monoid performs. For example, when using the sum_monoid to give numbers a monoid implementation, 0 is used. This is because

a + 0 = a
0 + a = a

and + is the definition of append / operator^ for sums.

The monoid operation itself.

While implementations should follow the general style of this interface, they are also allowed to accept for instance const M&, but never a non-const reference or other type that would allow mutation of either m1 or m2.
