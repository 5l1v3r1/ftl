Convenience operator to ease applicative style programming.

a * b <=> applicative<F>::apply(a, b)

Where a and b are complete types of F and F is an applicative functor.

Variable Documentation

constexpr aApply ftl::aapply {}

Compile time instance of aApply.

Makes it syntactically cheap to use applicative::apply in higher- order functions and such.

Example:

template<typename F, typename A, typename B>
auto foo(const A& a, const B& b) -> ftl::result_of<F(A,B)>;
AnApplicative<ftl::function<U,T>> ff = ...;
AnApplicative<T> ft = ...;
foo(aapply, ff, ft);
