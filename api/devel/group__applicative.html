Detailed Description

One step above a functor and one step below a monad.

#include <ftl/applicative.h>

What this means is that it has slightly more structure than a plain functor, but less than a monad. Specifically, what applicative adds on functors is a means of contextualising a "pure" value (go from type to F<type>, where F is some applicative functor), as well as a means to apply a function that is already in the context of the applicative functor.

All Applicative Functors are also Functors. There is no need to define an instance for both concepts.

Finally, an applicative instance must satisfy the following laws:

  • Identity law

    given

    template<typename T>
    T id(T t) { return t; }

    then,

    pure(id<T>) * v <=> v
  • Homomorphism law
    pure(f) * pure(x) <=> pure(f(x))
